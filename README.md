# 概念透镜组 (Concept Lens Array)

> **模糊的灵感是廉价的，清晰的约束才是昂贵的。**

一个三镜片认知系统，把"高端文案""看起来很酷的后台""要戳心的内容"这类模糊灵感，变成可执行、有边界、有验收标准的工程草案。

---

## 它解决什么问题？

你跟 AI 说"帮我写一个高端的护肤品文案"——AI 脑补了一堆华丽词藻，你改了八遍还是"感觉不对"。

**根因不是 AI 笨，是"高端"这个词本身就是黑洞。** AI 听到后自由发挥，你每次修改都在打补丁。

概念透镜组做的事：**在动工之前，把模糊形容词翻译成工程参数，用不可能三角强制取舍，锁定唯一核心任务。**

---

## 三镜片模型

```
模糊概念 ──→  ┌─────────┐     ┌─────────┐     ┌─────────┐  ──→ 可执行草案
               │  雾 镜  │ ──→ │  棱 镜  │ ──→ │ 聚焦镜  │
               │ (显影)  │     │ (分光)  │     │ (收敛)  │
               └─────────┘     └─────────┘     └─────────┘
               形容词→参数      识别不可能三角    锁定唯一任务
               "高端"=慢语速    深度/传播/篇幅    舍弃清单+验收标准
               +低密度+温情绪   三选二           +交付形态确认
```

| 镜片 | 核心任务 | 输入 | 输出 |
|:-----|:---------|:-----|:-----|
| **雾镜（显影）** | 把形容词翻译成工程参数 | 类比画面、使用场景、厌恶反例 | 《概念翻译草案》 |
| **棱镜（分光）** | 识别物理边界，强制取舍 | 草案 + 不可能三角选择 | 《边界确认书》 |
| **聚焦镜（收敛）** | 锁定唯一核心任务和交付形态 | 边界确认 + 交付形态选择 | 《最终参数确认表》 |

---

## 快速示例

**输入**：
```
帮我写一个高端的护肤品文案
```

**雾镜校准** — 把"高端"参数化：

| 参数 | 值 |
|:-----|:---|
| 语速 | 慢 |
| 信息密度 | 低 |
| 情绪温度 | 温 |
| 句式风格 | 中长句 |
| 禁用 | "震惊""三天见效" |

**棱镜分光** — 不可能三角：

```
深度洞察 / 传播性 / 篇幅短  →  三选二

用户选择：深度洞察 + 传播性 → 放弃篇幅短
```

**聚焦镜收敛** — 锁定唯一任务：

```
核心任务：写一段能戳中护肤人群深层焦虑的中长文案
交付形态：一段示例文案（200-300字）
舍弃清单：不做多版本对比、不做数据支撑、不做产品硬广
验收标准：用户看完会想"这就是我"
```

## 与 grill-me 的区别

grill-me 是 GitHub 上 8.9 万 Star 的需求追问工具。两者**不是竞品，是上下游**：

| 维度 | grill-me | concept-lens-array |
|:-----|:---------|:-------------------|
| 体积 | 147 bytes / 5 行 | 1061 行 + 示例 |
| 核心动作 | 决策树遍历追问 | 三镜片递进校准 |
| 对模糊的态度 | 消除——问完每个分支 | 结构化——把模糊变成参数 |
| 取舍机制 | 无 | 不可能三角强制三选二 |
| 边界标注 | 无 | 显式声明 AI 能/不能做什么 |
| 输出物 | Q&A 对话流（需二次整理） | 三份结构化文档（可直接执行） |
| 最佳场景 | 工程/编程需求 | 模糊概念落地 |

**组合用法**：模糊概念 → concept-lens-array（定"做什么+不做什么"）→ grill-me（追问"每个分支怎么做"）→ 执行

## 适用场景

- **文案创作**：从"要戳心"到"具体洞察 + 句式结构"
- **市场 BP 分析**：从"要全面"到"核心假设 + 验证路径"
- **Skill 设计**：从"要智能"到"输入输出规范 + 边界条件"
- **灵感转化**：从"有个点子"到"可行性第一步动作"
- **Vibe Coding 预处理**：在打开 Claude Code 之前，先把"看起来很高级的后台"变成参数表

## 安装

```bash
git clone https://github.com/Liangyuan-create/concept-lens-array.git
cp -r concept-lens-array/ /path/to/your/skills/user/
```

## 使用方式

在 AI 对话中输入以下任一触发词：

```
我有一个想法 / 我有一个概念 / 我有一个需求
启动概念透镜组 / 概念校准 / 透镜校准
```

或直接描述你的模糊需求：
```
帮我做一个看起来很高级的实验室数据管理系统
```

AI 会自动进入三镜片流程，按顺序执行：雾镜校准 → 棱镜分光 → 聚焦镜收敛。

---

## 核心原则

1. **严禁直接给结果** — 必须按三镜片流程逐步执行
2. **不可跳步** — 每个镜片必须完成才能进入下一个
3. **明确边界** — 温柔但坚定地指出 AI 能力边界
4. **完成割舍** — 用不可能三角逼用户做选择，舍弃清单一旦确认不再追加
